# 家庭生活记录工具 PRD

## 1. 产品定位

本项目是一个本地自用、私有化、轻量的家庭生活记录系统，用来替代 Synology Note Station 在家庭琐事记录场景中的部分能力。

它不是通用商业笔记软件，而是面向家庭长期记录的生活档案工具，重点解决快速记录、结构化整理、后期检索、历史导入和本地备份。

## 2. 核心目标

1. 支持导入 Synology Note Station 导出的历史记录。
2. 新建记录足够快，适合安卓手机端随手记录。
3. 通过分类、标签、来源、时间等字段降低流水账感。
4. 首页以生活时间线为主，而不是传统文档列表。
5. 支持标题、正文、分类、标签和时间范围检索。
6. 数据尽量本地化，方便整体备份到 NAS。
7. 项目结构清楚，方便后续迭代。

## 3. MVP 功能范围

### 首页 / 时间线

- 按时间倒序展示所有记录。
- 每条记录显示标题、摘要、时间、分类、标签。
- 支持快速搜索。
- 支持按分类筛选。
- 支持按标签筛选。

### 快速新建记录

- 手机端底部固定“新建”按钮。
- 新建时可选择记录类型：
  - 普通记录
  - 家庭事务
  - 维修维护
  - 购物消费
  - 账号资料
  - 重要备忘
  - 临时想法
- 支持标题、正文、分类、标签、时间、附件。
- 标题为空时，系统使用正文前几十个字生成标题。

### 分类体系

默认分类：

- 家庭事务
- 房屋 / 设备
- 维修 / 售后
- 购物 / 消费
- 证件 / 账号
- 孩子 / 教育
- 老人 / 健康
- 宠物
- 工作 / 杂事
- 临时记录
- 未分类

### 标签系统

- 一条记录可以有多个标签。
- 标签用于跨分类检索。
- 常见标签包括：水电、宽带、物业、发票、保修、医院、学校、车辆、家电、密码、NAS。
- 支持标签管理。

### 历史数据导入

- 支持导入 Synology Note Station 导出的文件。
- 不假设导出格式固定，先建设可扩展导入解析模块。
- 收到实际样例后，先分析文件结构，再写解析逻辑。
- 导入保留原始标题、正文、创建时间、更新时间、附件、原始路径、原始分类等信息。
- 导入前支持预览。
- 支持导入失败记录查看。

### 详情页

- 查看完整记录内容。
- 支持编辑。
- 显示记录来源：手动创建 / Note Station 导入。
- 显示创建时间、更新时间、分类、标签。
- 后续可扩展关联记录。

### 搜索

- 支持标题和正文全文搜索。
- 支持分类、标签、时间范围筛选。
- 搜索结果适合手机端阅读。

### 数据备份

- 数据库存储在 `data/app.db`。
- 附件存储在 `data/attachments/`。
- MVP 支持导出 JSON。
- 后续扩展导出 Markdown。

## 4. 非目标

MVP 暂不做：

- 多用户协作。
- 复杂权限系统。
- 第三方云同步。
- 富文本编辑器深度能力。
- 自动 AI 分类。
- 跨设备实时同步。

## 5. 技术方案

- 前端：React + Vite
- 后端：Express
- 数据库：SQLite
- 附件：本地文件夹
- 数据目录：项目下 `data/`
- 搜索：MVP 先使用 SQLite LIKE，后续升级 FTS5
- 部署：本地电脑优先，后续支持 NAS / Docker

## 6. 页面结构

- `/`：首页时间线
- `/search`：搜索页
- `/notes/new`：新建记录
- `/notes/:id`：详情页
- `/notes/:id/edit`：编辑页
- `/categories`：分类管理
- `/tags`：标签管理
- `/import`：导入入口
- `/import/:batchId`：导入预览 / 结果
- `/settings`：设置 / 备份

## 7. 数据结构草案

核心表：

- `notes`
- `categories`
- `tags`
- `note_tags`
- `attachments`
- `import_batches`
- `import_items`

后续可扩展：

- `note_links`
- `settings`
- `backup_logs`

## 8. 开发阶段

1. 项目骨架：前端、后端、SQLite、README、PRD。
2. 记录闭环：首页、新建、详情、编辑。
3. 搜索筛选：标题、正文、分类、标签。
4. 附件能力：上传、本地保存、详情展示。
5. 导入框架：批次、预览、失败记录、中间格式。
6. Note Station 样例解析：基于实际导出文件实现 parser。
7. 备份导出：JSON 导出和备份说明。
8. 手机端打磨：安卓优先优化交互和视觉。
