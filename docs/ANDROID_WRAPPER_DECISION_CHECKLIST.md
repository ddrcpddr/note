# Android 封装决策清单

本清单用于真正创建 Android 工程前的最后确认。当前不填写真实 NAS 地址、账号、密码、token、签名密钥，也不把任何 keystore 提交到 Git。

## 1. 推荐默认选择

| 项目 | 推荐值 | 原因 |
| --- | --- | --- |
| 封装路线 | Android WebView 壳 | 适合家庭局域网 / NAS 应用，不要求公网 HTTPS 或应用商店资产校验 |
| 数据位置 | 继续保存在 NAS / Express / SQLite | 保持当前家庭集中存储模型，不引入手机本地同步冲突 |
| App 名称 | 家事记 | 与当前 PWA manifest 和 V1 视觉一致 |
| 包名 | 待用户确认 | 包名一旦用于发布和签名，不宜随意更改 |
| NAS 地址策略 | 首次打开由用户输入家庭服务地址 | 避免把真实 NAS 地址写入仓库或 APK 默认配置 |
| 访问口令 | 复用现有 `NOTE_ACCESS_PIN` | 不新增账号系统，保持家庭自用轻量边界 |
| 离线能力 | 暂不做复杂离线同步 | 当前数据源在 NAS，离线同步属于后续大架构 |
| 签名 | 先调试签名，发布签名由用户决定 | 避免在仓库中产生或保存敏感签名文件 |

## 2. 用户需要确认

真正开始 Android 工程前，请确认：

1. 包名，例如 `com.yourname.home_note`。
2. App 显示名是否继续使用“家事记”。
3. 封装路线是否采用“原生 Android WebView 壳”。
4. 最低 Android 版本，例如 Android 8、10 或 12。
5. NAS 地址是否采用“首次打开手动输入”。
6. 是否需要记住上次输入的服务地址。
7. 是否只做家庭自用 APK。
8. 是否允许本地生成调试 keystore。
9. 是否已有发布签名；如果有，不要发到聊天或提交到仓库。
10. 是否允许引入 Android 工程目录和 Gradle wrapper。

## 3. 不建议现在做的选项

- 不建议把真实 NAS 地址写死进代码。
- 不建议把 SQLite 数据库复制到手机本地。
- 不建议做双向同步、冲突合并和离线编辑。
- 不建议现在做复杂账号 / 权限系统。
- 不建议在没有 HTTPS 域名和 assetlinks 的情况下优先做 TWA。
- 不建议提交 keystore、签名密码、NAS 密码、token 或任何真实地址。

## 4. WebView 壳最小功能范围

如果采用推荐路线，第一版 Android 壳只做：

- 启动页检查是否已有家庭服务地址。
- 没有地址时显示一个本地配置页，输入 `http://<局域网IP或NAS地址>:3300` 这类地址。
- 使用 WebView 打开该地址。
- 支持文件选择器，用于现有附件上传。
- 支持返回键：WebView 可返回时返回上一页，否则退出 App。
- 支持清除 / 修改服务地址。
- NAS 不可访问时显示可理解的错误页。

不做：

- 本地笔记数据库。
- 离线新建记录。
- 自动发现 NAS。
- 推送通知。
- 后台同步。
- 应用商店发布流程。

## 5. Android 工程启动验收

创建 Android 工程后的第一阶段验收：

1. APK 可以在一台 Android 手机上安装。
2. 首次打开可以输入服务地址。
3. 输入家庭局域网地址后能打开首页。
4. 首页、详情、新建、搜索、分类、设置可访问。
5. 附件文件选择器可唤起系统文件选择。
6. `NOTE_ACCESS_PIN` 开启时可解锁。
7. 返回键行为可接受。
8. 断开 NAS 服务后错误提示能看懂。
9. 仓库没有新增真实 NAS 地址、keystore、密码、token、数据库、备份、导出或附件。
10. `npm.cmd run check`、`npm.cmd run test`、`npm.cmd run build` 仍通过。

## 6. 当前暂停点

在用户确认第 2 节之前，不创建 Android 工程、不安装 Android 依赖、不生成 keystore。下一步应先完成真实 Android 手机 Web / PWA 试运行，再按本清单启动封装。
