# 手机端家庭试运行验收清单

本清单用于我和家人在安卓手机上验收家庭局域网 / NAS 试运行版本。当前阶段不继续扩展新功能，重点确认恢复后的正式 SQLite 数据库、Docker 真实 `data/` 挂载和移动端 PWA 使用是否稳定。

## 当前前提

- 正式数据库已经从健康备份恢复，`npm.cmd run check` 显示 `integrityCheck: "ok"`。
- 当前恢复后的正式库记录数为 `111` 条；损坏库中额外 1 条记录不作为可信数据源。
- Docker 使用默认 `./data:/data` 挂载真实数据目录，`NOTE_DATA_DIR=/data`。
- `docker compose up -d` 后本机地址为 `http://127.0.0.1:3300/`。
- 手机访问时应使用当前机器或 NAS 的局域网地址，例如 `http://<电脑或NAS局域网IP>:3300/`。
- 不要把真实局域网 IP、NAS 地址、账号、密码或 token 写入 Git 文档或配置。

## 试运行前准备

1. 确认服务已经在本机、NAS 或 Docker 容器中启动。
2. 确认手机与服务所在机器处于同一 Wi-Fi / 局域网。
3. 在电脑上确认：

```bash
npm.cmd run check
npm.cmd run smoke -- --base-url http://127.0.0.1:3300
```

4. 如果手机打不开电脑地址，优先检查：

- Windows 防火墙是否允许 Docker Desktop 或端口 `3300` 入站访问。
- 手机是否和电脑在同一个 Wi-Fi。
- 电脑当前局域网 IP 是否变化。
- 路由器是否开启了 AP 隔离 / 访客网络隔离。

5. 试运行前按 `docs/BACKUP_RESTORE_DRILL.md` 保存数据库、附件目录和 NAS 快照。

## 安卓手机验收流程

| 序号 | 流程 | 操作 | 通过标准 | 结果 |
| --- | --- | --- | --- | --- |
| 1 | 手机浏览器打开局域网地址 | 在 Chrome / Edge 输入 `http://<局域网IP>:3300/` | 首页能打开，不白屏 | 待人工验收 |
| 2 | 添加到桌面 / PWA | 浏览器菜单选择“添加到主屏幕”或“安装应用” | 桌面出现应用图标，点击后可打开 | 待人工验收 |
| 3 | 首页查看记录 | 打开首页，向下滚动记录列表 | 能看到当前正式库记录和导入记录 | 待人工验收 |
| 4 | 新建一条记录 | 点击加号，选择“我”或“爱人”，填写标题和正文并保存 | 保存后进入详情或列表可见 | 待人工验收 |
| 5 | 刷新后记录不丢失 | 浏览器刷新页面或重新打开 PWA | 刚才新建的记录仍存在 | 待人工验收 |
| 6 | 搜索刚才新建记录 | 搜索标题或正文关键词 | 搜索结果能找到该记录 | 待人工验收 |
| 7 | 查看 Note Station 导入记录详情 | 搜索或从“未分类 / 待整理”进入一条导入记录 | 能看到标题、正文、来源、时间、原始路径和附件元数据 | 待人工验收 |
| 8 | 分类筛选 | 打开分类页，进入“未分类 / 待整理”或其他分类 | 能看到对应记录，页面不跳错 | 待人工验收 |
| 9 | 成员筛选 | 在首页或搜索页切换“全部 / 我 / 爱人” | 筛选结果符合当前成员 | 待人工验收 |
| 10 | 设置页手动备份 | 进入设置，点击“立即备份” | 页面提示成功，`data/backups/` 下出现备份文件 | 待人工验收 |
| 11 | JSON 导出 | 设置页点击“导出 JSON” | 页面提示成功，`data/exports/` 下出现 JSON 文件 | 待人工验收 |
| 12 | Markdown 导出 | 设置页点击“导出 Markdown” | 页面提示成功，`data/exports/` 下出现 Markdown 文件 | 待人工验收 |
| 13 | 上传小附件 | 新建记录时选择一张小照片或文件 | 保存成功，文件落到 `data/attachments/`，详情页显示附件元数据 | 待人工验收 |
| 14 | PIN 开启 / 关闭 | 如配置 `NOTE_ACCESS_PIN`，手机端打开后输入口令；未配置时不应出现锁屏 | PIN 开启时可解锁，关闭时不影响访问 | 待人工验收 |
| 15 | 存储目录探测 | 设置页点击“检查当前数据目录” | database、attachments、backups、exports 均显示可写 | 待人工验收 |
| 16 | NAS 离线或服务不可用提示 | 临时停止服务或访问错误地址 | 提示能看懂，不误导为已保存 | 待人工验收 |
| 17 | 长标题 / 长正文 / 长链接 | 打开含长 URL 或长正文的记录 | 不横向溢出，不遮挡按钮 | 待人工验收 |
| 18 | 底部导航遮挡检查 | 首页、搜索、分类、设置页滚动到底部 | 底部导航不挡住关键按钮和正文 | 待人工验收 |
| 19 | 添加到桌面后的启动体验 | 从桌面图标启动应用 | 首屏能正常加载，返回/刷新体验可接受 | 待人工验收 |

## 建议记录的问题

人工验收时如果发现问题，建议记录：

- 手机型号和浏览器。
- Android 版本。
- 访问方式：浏览器 / PWA 桌面图标。
- 出问题的页面。
- 操作步骤。
- 预期结果和实际结果。
- 是否能稳定复现。
- 是否影响数据保存。
- 页面截图，截图中不要包含真实隐私正文。

更完整的问题记录模板见后续 `docs/TRIAL_FEEDBACK_TEMPLATE.md` 和 `docs/TRIAL_FEEDBACK_LOG.md`。

## 当前不在本轮验收范围

- 新增成员。
- 复杂权限系统。
- 外网访问、内网穿透或反向代理。
- 安卓原生 App 封装。
- 复杂离线同步。
- 从损坏库抢救额外 1 条记录。
