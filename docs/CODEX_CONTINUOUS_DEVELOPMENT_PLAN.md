# Codex 持续开发工作计划：家庭生活记录工具

更新时间：2026-07-03

本项目是用户个人和家庭自用工具，不再按复杂商业发布流程推进。之前围绕 RC1 / RC2、Gate、试运行发布清单的流程暂停，不作为当前主线任务。

本文件用于约束 Codex 后续工作，避免再次把时间消耗在旧的 RC / 发布流程上。

## 1. 项目定位

note 是一个部署在家庭 NAS / 局域网服务器上的家庭生活记录工具，用来替代 Synology Note Station 的家庭使用场景。

项目特点：

- 个人和家庭自用。
- 手机端优先。
- 数据保存在自己的电脑 / NAS / Docker 数据目录。
- 重点是日常记录、历史导入、搜索、分类、备份、导出。
- 不按商业产品发布节奏管理。

项目不是：

- 企业后台。
- 商业协作平台。
- SaaS 发布项目。
- 复杂权限系统。
- Android 优先项目。

## 3. 当前第一优先级：富文本编辑

目标：让家庭记录具备稳定的富文本编辑能力。

必须支持：

- 新建记录时使用富文本。
- 编辑已有记录时使用富文本。
- 详情页展示富文本。
- 兼容现有纯文本记录。
- 兼容 Note Station 导入的 HTML / 富文本内容。
- 不破坏搜索、分类、标签、成员、备份、JSON 导出、Markdown 导出和附件功能。

执行原则：

- 先写方案，再小步实现。
- 不直接开发完整复杂块编辑器。
- 不全量迁移数据库。
- 不破坏 `notes.content` 纯文本 fallback。
- 不把未清理 HTML 原样插入页面。
- 不引入大型依赖，除非先说明理由并等待用户确认。

建议下一步：

```text
富文本编辑专项 - 数据兼容方案 + 最小编辑体验实现计划
```

## 4. 第二优先级：富文本页面收口

只围绕富文本影响的页面做收口：

- 新建记录页。
- 编辑记录页。
- 详情页。

原则：

- 保持当前移动端风格。
- 不重做整套 UI。
- 不新增无关功能。
- 工具栏轻量。
- 长正文、链接、列表、附件入口在手机端可用。

## 5. 第三优先级：Product Design 7 图最终视觉还原

Product Design 7 张最终图仍然重要，是 Android 打包前必须完成的视觉基准。

但当前暂停，不阻塞富文本开发。

后续进入时必须遵守：

- 只按 Product Design 7 张最终图和 Figma 实现规格还原。
- 不凭感觉微调 CSS。
- 不重新设计。
- 不调用 Product Design 生成新方案。
- 不生成新图片。
- 每次只处理 1 个页面或 1 组强相关页面。
- 必须用 390px / 430px 截图验收。

## 6. 第四优先级：Android 封装

Android / WebView / TWA 排最后。

进入前必须完成：

1. 富文本编辑稳定。
2. 富文本相关页面收口。
3. Product Design 7 图最终视觉还原。
4. 用户确认包名、App 名称、最低 Android 版本、NAS 地址配置方式、签名方式和封装路线。

当前不要创建 Android 工程，不生成 APK，不引入 Android 依赖。
