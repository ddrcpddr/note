# Trial Feedback Template

本模板用于家庭成员在真实手机、家庭局域网、NAS / Docker 试运行期间记录问题。请尽量按结构填写，避免只留下“点了没反应”这类难以复现的信息。

## 使用原则

- 只记录真实试运行中遇到的问题、疑问或体验不顺。
- 不在本文件中写入真实 NAS 地址、账号、密码、token、家庭隐私正文或完整 Note Station 原文。
- 截图路径只写本地临时路径或脱敏后的文件名，不提交真实截图到 Git。
- 每个问题尽量单独一条反馈，方便一个 bug 一个 commit。
- 如果涉及数据丢失、数据库错误、备份失败，优先标为 P0 / P1，并停止继续大量操作。

## 严重程度

| 等级 | 含义 | 示例 |
| --- | --- | --- |
| P0 | 数据风险或核心流程完全不可用 | 数据库损坏、保存后记录丢失、备份恢复失败 |
| P1 | 常用流程明显不可用 | 新建失败、详情打不开、搜索完全不可用、按钮点了无反应 |
| P2 | 影响体验但有绕路 | 文案误导、布局遮挡、筛选结果不符合预期 |
| P3 | 轻微优化建议 | 间距、字号、表达习惯、图标偏好 |

## 反馈条目模板

```markdown
### YYYY-MM-DD HH:mm - 简短标题

| 字段 | 内容 |
| --- | --- |
| 日期 | YYYY-MM-DD HH:mm |
| 测试人 | 我 / 爱人 / 家人代号 |
| 设备型号 | 例如 Android 手机型号，避免写隐私序列号 |
| Android 版本 | 例如 Android 14 |
| 浏览器 | Chrome / Edge / 系统浏览器 / PWA |
| 访问方式 | 浏览器 / 添加到桌面 PWA / Docker 本机 / NAS |
| 页面 | 首页 / 新建 / 详情 / 搜索 / 分类 / 设置 / 导入 Note Station / 成员管理 |
| 操作步骤 | 1. ... 2. ... 3. ... |
| 预期结果 | 应该发生什么 |
| 实际结果 | 实际发生什么 |
| 截图路径 | 本地临时路径或脱敏截图文件名；不要提交真实隐私截图 |
| 是否影响数据 | 是 / 否 / 不确定 |
| 严重程度 | P0 / P1 / P2 / P3 |
| 是否已复现 | 是 / 否 / 待复现 |
| 是否已修复 | 否 / 是，commit `xxxxxxx` |
| 对应 commit | 修复前基线或修复 commit |

补充说明：
- 可写脱敏摘要，不写完整真实笔记正文。
```

## 手机试运行建议最小复现信息

如果时间很赶，至少提供：

```text

预期：
实际：
截图：
是否影响数据：
```
