# 下一步

更新时间：2026-07-03

当前项目是个人和家庭自用的生活记录工具，不再围绕复杂发布流程消耗时间。当前主线保持为：

```text
富文本编辑 -> 富文本相关页面收口 -> Product Design 7 图最终视觉还原 -> Android 封装
```

## 当前已完成：富文本编辑能力

已完成日常可用的第一版富文本编辑：

- 新建记录页可以输入富文本。
- 编辑记录页可以编辑富文本。
- 详情页可以安全展示富文本，并保留纯文本切换。
- 旧纯文本记录继续正常显示、搜索和编辑。
- Note Station 导入记录的原始 HTML 仍可安全展示，用户编辑后的版本保存到独立富文本字段。
- 搜索仍基于纯文本。
- JSON 导出和 Markdown 导出保持可用。
- 不引入大型编辑器依赖。

## 下一步优先级 1：富文本相关页面收口

建议先进行真实手机上的日常试用，而不是马上扩展复杂功能。重点检查：

- 新建页工具栏在手机宽度下是否顺手。
- 旧记录编辑后是否符合预期。
- Note Station 导入记录编辑后是否保留了需要的内容。
- 长正文、列表、链接、换行是否稳定。
- JSON / Markdown 导出是否符合家庭备份习惯。

只做小修：按钮间距、工具栏横向滚动、长内容换行、详情页格式展示等。不要重做整套 UI。

## 下一步优先级 2：富文本能力增强候选

这些不是马上要做的内容，只有日常试用确认需要后再排：

- 正文内图片插入与附件路径映射。
- 更好用的链接编辑体验。
- 任务列表。
- 更完整的 HTML 转 Markdown。
- 富文本粘贴清理增强。
- 工具栏状态高亮。

暂不做：表格、多人协同、块拖拽、AI 整理、复杂模板。

## 下一步优先级 3：最终视觉还原

Product Design 7 张最终图仍然重要，但不阻塞富文本试用。后续进入 Android 前，需要按 Figma 实现规格逐页还原：

- 首页
- 新建记录
- 记录详情
- 搜索
- 分类
- 导入 Note Station
- 设置 / 备份

执行规则保持不变：每次只修 1 个页面或 1 组强相关页面，用 390px / 430px 截图验收，不凭感觉全站微调。

## 下一步优先级 4：Android 封装

Android / WebView / TWA 继续排最后。进入前需要确认：

- 富文本编辑已稳定。
- 主要页面视觉还原已稳定。
- NAS 访问方式已稳定。
- App 名称、包名、图标、签名方式、NAS 地址配置方式已确认。

当前不要创建 Android 工程，不生成 APK。
