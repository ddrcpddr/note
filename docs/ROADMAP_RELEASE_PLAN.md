# 发布路线图

更新时间：2026-07-03

当前目标不是继续堆功能，而是把 note 推进到家庭 NAS 上可稳定试用的 RC1 状态，然后根据真实反馈决定下一轮大功能。

## 当前发布阶段

阶段：RC1 / 家庭局域网试运行版

RC1 的定义：

- 可以在本机和 Docker/NAS 环境启动。
- 当前正式数据库完整性通过。
- 核心记录、搜索、分类、成员、备份、导出、导入结果查看可用。
- 当前真实 Note Station 样例已导入。
- 手机浏览器可完成基本家庭记录流程。
- 不承诺完整富文本编辑、安卓原生、复杂权限、离线同步或高级附件管理。

## RC1 进入条件

- `npm.cmd run check` 通过。
- `npm.cmd run test` 通过。
- `npm.cmd run build` 通过。
- 数据库 integrity check 通过。
- Docker / HTTP smoke 至少在本机或可用环境验证一次。
- `data/`、数据库、附件、备份、导出、`.nsx`、日志没有进入 Git。
- README、PROJECT_STATUS、NEXT_STEPS、RUN_RESULT_HANDOFF、QA_REPORT_CURRENT 已同步。

## RC1 试运行验收

建议先由 1-2 位家庭成员在真实手机上试用 3-7 天，重点检查：

1. 手机浏览器打开局域网地址。
2. 首页能看记录，长标题不影响使用。
3. 新建一条真实家庭记录，刷新后不丢失。
4. 搜索刚才新建的记录。
5. 查看一条 Note Station 导入记录详情。
6. 分类筛选和成员筛选正常。
7. 设置页手动备份成功。
8. JSON / Markdown 导出成功。
9. NAS 重启或容器重启后数据仍在。
10. 家人是否觉得页面像家庭记录工具，而不是后台系统。

## RC1 后的真实大功能

### P1：NAS 运维增强

- 完善 Synology Container Manager / Docker Compose 操作手册。
- 明确数据目录挂载、备份目录、恢复流程。
- 增加健康检查和 smoke 命令的日常运行说明。
- 完善日志排查和快照策略。

### P1：导入后整理增强

- 增强 Note Station 导入记录的批量分类、批量标签、失败项查看。
- 给导入来源、原始路径、原始分类路径提供更清晰的整理入口。
- 对未知 `.nsx` 变体继续坚持 dry-run 优先。

### P1：附件管理增强

- 附件删除、替换、下载、预览。
- 附件与备份恢复的一致性检查。
- 大文件上传和移动端提示优化。

### P1/P2：富文本能力第二阶段

- 当前只完成安全只读渲染。
- 下一阶段可评估轻量编辑器，但必须先确认依赖、数据库兼容、导出策略和回滚方案。
- `notes.content` 继续作为纯文本搜索和 fallback，不轻易迁移。

### P2：Android 封装

- 优先验证 PWA 是否够用。
- 如果需要 Android，再确认 WebView / TWA、应用名、图标、签名、局域网配置和安装方式。
- 不在 RC1 前提前进入 Android。

### P2：访问保护增强

- 当前 `NOTE_ACCESS_PIN` 是简单访问口令，不是账号体系。
- 是否需要家庭成员登录、HTTPS、反向代理、局域网访问控制，需要试运行反馈后决定。

## 暂缓事项

- 不新增成员体系，当前只保留 `我 / 爱人`。
- 不做复杂权限系统。
- 不做完整富文本编辑器。
- 不做 Android 原生封装。
- 不重构数据库结构。
- 不重新设计 UI。
- 不凭感觉继续视觉微调；若继续视觉还原，必须按 Figma 实现规格逐页做。

## 停止条件

遇到以下情况必须先确认：

- 需要真实 NAS 地址、账号、密码或 token。
- 需要改变数据库结构或全量迁移。
- 需要引入大型依赖。
- 需要新增成员/账号/权限体系。
- 需要修改真实 Note Station 导入数据。
- 需要从损坏数据库中抢救未经确认的数据。
