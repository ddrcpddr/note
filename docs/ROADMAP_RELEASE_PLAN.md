# 开发路线图

更新时间：2026-07-03

当前项目定位已重新收口：note 是个人和家庭自用工具，不按商业产品发布节奏推进。复杂 RC1 / RC2、发布清单、试运行版本管理流程暂停，不再作为当前主线。

## 当前主线

```text
富文本编辑 -> 富文本页面收口 -> 最终视觉还原 -> Android 封装
```

## 已暂停的旧主线

以下内容不再作为当前优先级：

- RC1 / RC2 发布阶段管理。
- 复杂试运行验收清单扩写。
- 围绕发布流程反复生成交接文档。
- Android 封装前置准备。
- 继续视觉还原或 Figma 调用。
- 与富文本主线无关的新功能。

保留已有部署、备份、恢复、QA 文档作为参考，但不继续围绕它们消耗开发时间。

## Phase 1：富文本编辑能力

目标：补齐家庭记录最核心的正文编辑体验。

范围：

- 新建记录支持富文本。
- 编辑记录支持富文本。
- 详情页展示富文本。
- 兼容现有纯文本记录。
- 兼容 Note Station 导入 HTML / 富文本。
- 保持纯文本搜索能力。
- 保持 JSON / Markdown 导出可用。
- 保持备份、分类、标签、成员、附件功能不被破坏。

约束：

- 不做全量数据库重构。
- 不破坏 `notes.content` 作为纯文本 fallback。
- 不直接引入大型编辑器，除非先说明理由。
- 不把未清理 HTML 原样插入页面。
- 不在这一阶段重做 UI。

验收：

- 纯文本记录可以继续创建、编辑、搜索、导出。
- 富文本记录可以创建、编辑、详情展示。
- Note Station 导入的富文本内容可以继续安全展示。
- 恶意 HTML 会被清理。
- `npm.cmd run check/test/build` 通过。

## Phase 2：富文本相关页面收口

范围：

- 新建记录页
- 编辑记录页
- 详情页

目标：让富文本入口在手机端可用、轻量、不像后台编辑器。

原则：

- 工具栏只保留必要能力。
- 长正文不遮挡底部操作区。
- 附件入口不被富文本工具栏挤压。
- 保持当前移动端风格，不重做整套 UI。

## Phase 3：最终视觉还原

Product Design 7 张最终图仍是 Android 前必须完成的视觉基准，但不阻塞富文本开发。

进入条件：

- 富文本编辑稳定。
- 新建、编辑、详情页完成富文本收口。

执行方式：

- 逐页按 Product Design 7 张图和 Figma 实现规格还原。
- 不凭感觉微调。
- 不调用 Product Design 生成新方案。
- 不生成新图片。
- 每页用 390px / 430px 截图验收。

页面顺序建议：

1. 新建记录 / 编辑记录 / 详情页
2. 首页
3. 搜索
4. 分类
5. 导入 Note Station
6. 设置 / 备份

## Phase 4：Android 封装

Android 继续排最后。

进入条件：

- 富文本能力稳定。
- 最终视觉还原完成。
- 用户确认封装路线和基本信息。

当前禁止：

- 不创建 Android 工程。
- 不生成 APK。
- 不引入 Android / Capacitor / TWA 依赖。
- 不写入真实 NAS 地址、密码、token 或签名文件。

## 后续但非当前主线

这些能力可以保留为未来 P1/P2，但不抢富文本优先级：

- NAS 运维增强。
- 附件管理增强。
- 导入后整理增强。
- 访问保护增强。
- 成员新增。
- 离线同步。

## 当前停止条件

遇到以下情况必须暂停并询问：

- 需要改变数据库结构。
- 需要引入大型编辑器依赖。
- 需要迁移真实 Note Station 导入数据。
- 需要修改真实 NAS 地址或账号密码配置。
- 需要创建 Android 工程。
- 需要重新启动 Product Design / Figma 设计流程。
