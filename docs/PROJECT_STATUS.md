## 产品定位

note 是一个部署在家庭 NAS 上的多人家庭生活记录系统，用来替代 Synology Note Station 的家庭记录场景。它面向家人手机访问，强调轻量、温和、手机端优先、数据保存在自己设备或服务器中。

它不是企业协同系统，不是后台管理系统，也不是公开 SaaS。

## 当前真实可用功能

- 移动端页面： 首页、分类、搜索、详情、新建/编辑、导入 Note Station、设置、成员管理。
- 记录持久化：SQLite 正式库保存记录、分类、标签、成员、附件元数据、导入元数据。
- 记录操作：新建、读取、详情查看、编辑、归档、删除。
- 筛选检索：关键词搜索、分类筛选、标签筛选、成员筛选、来源筛选。
- 默认成员：只内置 `我`、`爱人` 两个默认成员。
- 成员资料：支持切换当前成员，并对这两个默认成员改名、改头像、改颜色。
- 分类体系：保留 11 个默认分类，`uncategorized` 统一展示为 `未分类 / 待整理`。
- 标签：可选择预置标签，也可以新增自定义标签；标签允许为空。
- 附件：支持基础附件上传到 `data/attachments/`，数据库只保存附件元数据和相对路径。
- Note Station 导入：真实 `.nsx` 已完成结构分析、dry-run、sandbox 导入、正式导入、正式库保护备份和导入报告；当前样例导入已经完成。
- 导入后整理：支持查看导入来源、原始路径/分类路径，并可对未分类导入记录做批量整理。
- 富文本只读：详情页可对 Note Station 导入保留的原始 HTML / 富文本做安全只读渲染，并保留纯文本 fallback。
- 备份：设置页支持手动备份，服务端支持可配置的自动备份。
- 恢复：已有 `npm.cmd run restore-db`，支持 dry-run 和 `--confirm`，恢复前会保留当前库副本。
- 导出：支持 JSON 导出和 Markdown 导出。
- 访问保护：支持可选 `NOTE_ACCESS_PIN` 简单访问口令。
- PWA：已有基础 manifest、runtime 图标和移动端添加到桌面的准备。
- Docker / NAS：已有 Dockerfile、docker-compose.yml、`.dockerignore` 和 NAS 部署说明。
- 验证命令：项目已有 `npm.cmd run check`、`npm.cmd run test`、`npm.cmd run build`、`npm.cmd run smoke`。

## 当前仍然模拟或受限的功能

- 成员新增：当前不新增真实家庭成员，只保留默认 `我 / 爱人`；新增成员以后再做。
- 权限系统：没有家庭成员账号、角色权限、私密记录或多账号体系。
- 富文本编辑：当前只有富文本安全只读渲染，没有完整富文本编辑器。
- 离线同步：PWA 只做基础添加到桌面，不做复杂离线同步或冲突合并。
- Android 原生封装：尚未进入 Android WebView / TWA 封装阶段。
- 附件管理增强：已有基础上传与元数据，尚未完成附件删除、替换、批量管理、预览下载体验增强。
- Note Station 格式泛化：当前真实样例已验证，其他未知 `.nsx` 变体仍必须先 dry-run，不硬猜格式。
- NAS 运维增强：已有部署文档和备份恢复工具，但还需要真实 NAS 长时间试运行、快照策略和故障演练。
- 视觉最终还原：分类页已按 Figma 规格重建过；其余页面若继续要求 Product Design 7 图高保真，需要逐页按 Figma 规格重建，不再凭感觉微调。

- `data/` 运行数据

- 导出文件
- 附件文件
- `.nsx` 文件
- dry-run JSON
- 解压后的真实 Note Station 内容
- 日志文件
- 密码、token、真实 NAS 地址、账号信息
