## 当前结论

note 已具备家庭日常使用的主要能力：网页 / Docker 服务端、SQLite、富文本、图片和附件、真实 Note Station `.nsx` 导入、备份导出，以及可离线保存并在恢复连接后同步的 Android 应用。

项目是个人和家庭自用工具，不再围绕 RC1 / RC2 或商业发布流程推进。当前重点是解决真机和 NAS 长期使用中的真实问题，而不是继续扩展大功能。

代码、自动化测试、APK 包结构和最新 GHCR 镜像均已验证。Android 大数据同步后的随机闪退已完成针对性修复，但仍需在 vivo X300 Pro 和 Huawei P30 Pro 真机上最终确认。

## 产品定位

- 用于替代 Synology Note Station 的家庭记录场景。
- 数据保存在手机本地和家庭 NAS / Docker。
- 手机端优先，支持断网记录和恢复连接后同步。
- 适合家庭事务、维修、购物、账单、备忘和历史资料。
- 保持轻量、温和，不做企业后台或复杂协作系统。

## 当前真实可用功能

### 网页与 Docker 服务端

- 页面：首页、分类、搜索、详情、新建 / 编辑、Note Station 导入、设置、成员管理。
- 记录操作：新建、查看、编辑、归档、删除。
- SQLite 保存记录、分类、标签、成员、附件和导入元数据。
- 支持关键词、分类、标签、成员和来源筛选。
- 内置 11 个默认分类，并支持创建和修改自定义分类。
- `uncategorized` 显示为“未分类 / 待整理”。
- 支持预置标签、自定义标签和无标签记录。
- 默认成员只保留“我 / 爱人”，支持切换当前成员和调整显示资料。
- 可通过 `NOTE_ACCESS_PIN` 启用简单访问口令。
- 已有基础 PWA 支持。

### 富文本与附件

- 新建和编辑页已接入 Tiptap 富文本编辑器。
- 支持常用文字格式、标题、列表、待办、引用、链接、颜色、高亮和表格。
- 详情页安全展示清理后的 HTML，并保留纯文本降级。
- 搜索和列表摘要继续使用纯文本。
- 图片和附件从富文本编辑器内插入。
- 文件保存在 `data/attachments/`，数据库保存元数据和相对路径。
- 正文内图片和附件可内联展示，未内联附件保留下载入口。
- JSON 和 Markdown 导出兼容富文本字段。

### Note Station 导入

- 已基于真实 Synology Note Station `.nsx` 样例实现解析。
- 网页端支持上传 `.nsx`，采用异步后台解析，避免 NAS 大文件长请求超时。
- 支持 dry-run、预览、确认导入、导入前自动备份、失败项和导入摘要。
- 保留纯文本、安全 HTML、原始 HTML、原始路径、原始分类和来源。
- 图片和附件复制到数据目录，并尽量恢复原有格式和正文位置。
- 已支持解析任务恢复，以及旧附件 ID 与当前附件文件之间的修复。
- Android 可以选择 `.nsx`，但解析和正式导入仍依赖已连接的 Docker / NAS；暂不支持手机完全离线解析。

### 备份、恢复与部署

- 设置页支持手动备份和存储目录检查。
- 服务端支持可配置自动备份。
- `npm.cmd run restore-db` 支持 dry-run、确认恢复和恢复前副本保护。
- 支持 JSON 和 Markdown 导出。
- 已提供 Dockerfile、Compose 示例、GHCR 自动构建和 Synology / QNAP 部署说明。

## Android 当前状态

### 已实现

- 当前 APK 是 Capacitor 本地优先应用，内置 React 前端，不是只打开远程网址的几十 KB 空壳。
- UI 仍运行在 Android 系统 WebView 中，但数据使用本地 SQLite / App 私有存储。
- 本地库覆盖记录、分类、标签、成员、附件元数据和同步队列。
- 支持离线新建、编辑、归档、删除和重启后读取。
- 支持本地保存图片 / 附件，恢复连接后进入同步流程。
- 设置页可以填写 Docker / NAS 地址，地址只保存在手机本地。
- 支持待同步队列、失败重试和基础冲突保护。
- 已补旧 Android / HarmonyOS WebView 的兼容处理，以及富文本编辑器的纯文本降级。

### 最新闪退修复

此前 Android 同步或导入大量 Note Station 记录后可能随机闪退。主要风险是 `/api/app-data` 一次返回全部记录的完整富文本、原始 HTML 和图片引用，并再次写入手机快照，造成 WebView 和存储压力。

当前处理：

- 首页和同步列表只返回轻量记录。
- 完整富文本和附件在打开单条详情时按需读取。
- 已同步远程记录写入手机快照前移除大体积富文本字段。
- 未同步的本地富文本草稿继续保留完整内容。

该修复已通过自动测试、APK 验证和精确 GHCR 镜像验证，但仍需真机复测。

当前 APK：

```text
android/app/build/outputs/apk/debug/app-debug.apk
```

已验证大小约 `25.7 MB`，`nativeShellOnly=false`。

## 当前未完成或受限

- Android 不能完全离线解析和导入 `.nsx`。
- 同步冲突没有可视化比较或自动合并。
- 没有可靠的后台常驻同步。
- 大附件长期离线、断点续传和多设备下载仍需增强。
- 附件删除、替换、批量管理和跨设备缓存清理仍是基础状态。
- 不支持新增真实家庭成员，当前只维护“我 / 爱人”。
- 没有家庭账号、角色权限、私密记录或复杂登录体系。
- 未承诺兼容所有未知 Note Station 导出变体。
- 最终视觉高保真还原尚未全部完成。
- 真机错误日志采集不足；若仍闪退，必须抓取 `adb logcat` 或增加 App 内错误日志导出。
