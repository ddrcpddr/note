# 家庭生活记录工具移动端原型设计

## 1. 产品设计理解

这是一个家庭成员共同使用的生活记录工具。它的核心不是“写文档”，而是把家里长期发生的小事变成可搜索、可回顾、可备份的家庭生活时间线。

产品仍然要轻量、温和、私有化。多人使用只用于标记“谁记录了这件事”和支持成员筛选，不把产品做成复杂协同系统。

数据集中保存在家里的 NAS。手机端只是访问同一个 Web/PWA 应用，不优先做每台手机本地数据库同步。

## 2. 信息架构

底部导航：

- 首页
- 分类
- 搜索
- 设置

突出操作：

- 新建记录按钮固定在底部中央或右下角。

核心对象：

- 记录
- 成员
- 分类
- 标签
- 附件
- 来源
- 同步/保存状态
- 导入批次
- 备份记录

## 3. 页面清单

1. 首页 / 生活时间线
2. 快速新建记录页
3. 记录详情页
4. 搜索页
5. 分类 / 分类管理页
6. Note Station 导入页
7. 设置 / NAS 存储与备份页

## 4. 用户使用流程

### 日常记录

打开首页 -> 点击新建 -> 当前成员自动带入 -> 输入正文 -> 选择类型、分类、标签 -> 保存 -> 回到首页时间线。

### 查找记录

进入搜索 -> 输入关键词 -> 按分类、标签、成员、时间筛选 -> 查看结果卡片 -> 进入详情。

### 导入旧数据

进入导入页 -> 上传 Note Station 导出文件 -> 解析预览 -> 查看数量、附件、失败项 -> 确认导入 -> 生成导入结果。

### 备份

进入设置 -> 查看 NAS 存储状态 -> 手动备份数据库 -> 导出 JSON。

## 5. UI 风格

关键词：

- 手机端优先
- 简洁
- 温和
- 生活感
- 卡片式
- 清爽
- 易读
- 不像后台管理系统
- 不像传统文档列表
- 不信息过载

视觉参考：

- Apple Notes 的简洁留白
- Notion 的清爽层级

- 轻量生活记录 App 的温和感

## 6. 设计系统

### 颜色

- 页面背景：`#F6F7F3`
- 卡片背景：`#FFFFFF`
- 主色：`#3F7B63`
- 主色浅底：`#E6F0EA`
- 强调色：`#D99A5F`
- 主文字：`#202421`
- 次级文字：`#68716B`
- 弱文字：`#969E98`
- 边框：`#DDE5DD`
- 成功：`#3F7B63`
- 警示：`#D99A5F`
- 失败：`#B85C5C`

### 字体

- 中文优先，使用系统中文字体。
- 页面标题：24-28px，600。
- 分区标题：18-20px，600。
- 卡片标题：16-18px，600。
- 正文：15-16px，400。
- 辅助信息：12-13px。

### 圆角

- 页面卡片：12px。
- 输入框：10px。
- 小标签：999px。
- 底部新建按钮：999px。

### 间距

- 页面左右边距：16px。
- 卡片内边距：14-16px。
- 卡片间距：12px。
- 分区间距：20-24px。
- 底部导航高度：64-72px。

### 卡片规范

记录卡片包含：

- 顶部：分类、时间、创建人头像。
- 中部：标题、摘要。
- 底部：标签、附件标识、保存/同步状态。

卡片阴影极轻，主要依靠背景、边框和留白分层。

### 按钮规范

- 主按钮：蓝绿色底，白字。
- 次按钮：白底，浅边框。
- 新建按钮：底部悬浮，蓝绿色，圆角饱满。
- 危险按钮只在删除、清空导入记录等场景出现。

### 同步/保存状态

状态样式：

- 已保存：绿色小圆点 + “已保存”
- 保存中：灰色旋转点 + “保存中”
- 保存失败：红色提示 + “保存失败”
- 离线，待同步：橙色提示 + “离线，待同步”

## 7. 页面布局

### 页面 1：首页 / 生活时间线

顶部：

- 产品名：家事记 / 家庭记录。
- 搜索入口。
- NAS 连接状态小提示，可弱化显示。

筛选区：

- 分类快捷筛选胶囊。
- 成员筛选头像胶囊，例如“全部、我、爱人”。

主体：

- 按时间倒序展示记录卡片。
- 卡片显示标题、摘要、时间、分类、标签、创建人、附件和同步状态。

底部：

- 首页、分类、搜索、设置导航。
- 新建记录按钮突出。

设计原因：

首页要让用户快速知道“最近家里发生了什么”，所以用时间线而不是文档列表。成员头像让多人记录自然融入，而不是变成协同后台。

### 页面 2：快速新建记录页

顶部：

- 返回。
- 标题“新建记录”。
- 保存按钮。

成员信息：

- 显示“我正在记录”“爱人正在记录”或当前成员头像。

主体：

- 类型胶囊：普通记录、家庭事务、维修维护、购物消费、账号资料、重要备忘、临时想法。
- 标题输入，选填。
- 大正文输入区。
- 分类选择。
- 标签输入。
- 附件入口。

底部：

- 保存状态提示。

设计原因：

新建记录不能像复杂表单。正文是主角，标题、分类、标签都是辅助整理字段。

### 页面 3：记录详情页

顶部：

- 返回。
- 编辑按钮。

正文区：

- 标题。
- 完整正文。
- 标签。

信息区：

- 创建人。
- 创建时间。
- 更新时间。
- 最后编辑人。
- 来源：手动创建 / Note Station 导入。
- 同步状态。
- 附件。

关联记录：

- 预留“相关记录”区域，后续可根据标签、分类、时间关联。

设计原因：

详情页优先保证阅读体验。结构化信息放在正文下方，完整但不压迫。

### 页面 4：搜索页

顶部：

- 搜索框。

筛选区：

- 分类筛选。
- 标签筛选。
- 成员筛选。
- 时间范围筛选。

结果区：

- 使用和首页一致的记录卡片。

空状态：

- “还没有找到相关记录，可以换个关键词或减少筛选条件。”

设计原因：

搜索页不做高级后台筛选器，仍然保持手机端直接、轻量。

### 页面 5：分类 / 分类管理页

展示默认分类：

- 家庭事务
- 房屋 / 设备
- 维修 / 售后
- 购物 / 消费
- 证件 / 账号
- 孩子 / 教育
- 老人 / 健康
- 宠物
- 工作 / 杂事
- 临时记录
- 未分类

每个分类卡片显示：

- 分类图标。
- 分类名称。
- 记录数量。
- 最近更新时间。

设计原因：

分类页像家庭事项入口，不像后台菜单。记录数量帮助用户理解每类内容的积累情况。

### 页面 6：Note Station 导入页

步骤：

1. 上传导出文件
2. 解析预览
3. 确认导入
4. 导入完成

预览信息：

- 记录数量。
- 分类数量。
- 附件数量。
- 失败项数量。
- 导入执行成员。
- 数据将保存到 NAS 的提示。

安全提示：

- 不覆盖现有记录。
- 导入前可预览。
- 失败项可稍后处理。

设计原因：

导入旧数据有风险感，页面要明确“先预览、再确认、可追溯”，让用户放心。

### 页面 7：设置 / NAS 存储与备份页

模块：

- 当前 NAS 连接状态。
- 当前数据存储位置。
- 附件存储位置。
- 最近备份时间。
- 手动备份按钮。
- 导出 JSON 按钮。
- 导出 Markdown，灰显，标注后续支持。
- 数据库位置说明。
- 附件目录说明。

设计原因：

设置页不做复杂系统配置，重点服务家庭长期保存和备份。

## 8. 转代码注意事项

1. MVP 使用集中式 NAS 部署，不要先做手机端本地数据库同步。
2. 记录表从一开始保留 `created_by_member_id`、`last_edited_by_member_id`、`sync_status`、`version`。
3. 附件存文件夹，数据库只存路径和元数据。
4. 首页卡片组件要支持成员头像、附件、同步状态。
5. 搜索接口要预留成员筛选。
6. 设置页要优先实现 NAS 路径展示、手动备份、JSON 导出。
7. Note Station 导入记录要能归属到执行导入的当前成员，并保留来源元数据。
8. 多人编辑冲突先用乐观锁提示，不做复杂合并。
