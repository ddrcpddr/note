# Figma Implementation Specs

本文件给前端 Codex 实现移动端界面使用，不是设计说明。视觉验收标准是 `design/home-records-prototype/` 中 7 张 Product Design 最终页面图，以及当前 Figma 中已重建的 7 个可编辑 Frame。

不要重新发散风格，不要使用整页截图实现 UI，不要把 Figma 大画布横向排列当成前端布局。每个 Frame 对应一个独立移动端页面 / 路由。

## 10. 内容显示和截断规则

- 分类标题必须完整显示，不允许省略号，不允许竖排文字。
- 记录卡片标题最多 2 行；超过后可截断，但不能影响卡片底部元信息。
- 记录摘要最多 2 行。
- 搜索结果标题最多 2 行。
- 设置项标题必须完整显示；说明文字最多 2 行。
- 文件名最多 1 行，可中间截断，但不能显示真实隐私文件名。
- Note Station 原始路径只能作为弱展示，不作为卡片主标题。
- 默认成员只内置 `我 / 爱人`；不要把 `老婆 / 妈妈 / 爸爸` 写入默认 UI。

## 11. 前端实现注意事项

- 不要把多个 Figma Frame 并排实现到一个页面。
- 不要使用整页截图替代 UI。
- 不要生成新视觉风格。
- 不要引入后台管理、企业协作、服务器控制台风格。
- 示例内容必须脱敏。
- 不要使用真实 NAS 地址、账号、密码、token 或真实 Note Station 笔记正文。
- 前端运行时图片优先使用 `design/image-assets/v1/runtime/` 下的小图；Figma/source 图只作为设计参考。
