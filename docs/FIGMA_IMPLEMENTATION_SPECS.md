# 家事记 App UI 设计规范

> 用途：供 Figma 重建、Codex 后续开发、产品交接使用。
> 原则：本规范锁定当前「家事记」移动端 UI 风格。后续只允许接入功能逻辑，不应随意重设计视觉样式。

---

## 1. 产品定位

**产品名称**：家事记
**产品类型**：家庭生活记录管理 App
**核心场景**：记录家中大小事、维修售后、购物消费、证件账号、老人健康、孩子教养、临时备忘、导入旧笔记与数据备份。

**设计关键词**：
- 温和、可信赖
- 家庭感、轻工具感
- 中文移动端原生体验
- 卡片式信息整理
- 低干扰、易扫读

linear-gradient(135deg, #D4EDE1 0%, #BDD8E8 100%)

## 8. 开发约束

### 8.1 必须保留的文件

```text
src/app/App.tsx
src/styles/theme.css
src/styles/fonts.css
AGENTS.md
```

### 8.2 后续开发规则

1. 新增功能逻辑时，只改 state、数据结构、事件处理和 API 接入。
2. 不要随意改颜色、字号、圆角、阴影和布局节奏。
3. 新增页面时，优先复用现有组件：
   - `Pill`
   - `CatIcon`
   - `RecCard`
   - `SubHeader`
   - `BottomNav`
4. 新增记录数据时，扩展 `ALL_RECS` 或后续替换为真实数据源。
5. 不引入新的 UI 组件库。
6. 不修改 Tailwind token 名称。
7. 不删除 `@theme inline` 映射。

### 8.3 推荐技术栈

- React 18 + TypeScript
- Tailwind CSS
- lucide-react 图标
- Noto Sans SC 中文字体
