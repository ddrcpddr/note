# 产品设计上下文

## 1. 产品定位

这是一个部署在家庭 NAS 上的多人家庭生活记录系统，用来替代 Synology Note Station。

家庭成员可以通过手机浏览器或 PWA 访问，快速记录家庭事务。所有记录、附件、导入数据统一保存到家里的 NAS，方便集中备份、统一管理和长期保存。

## 2. 不是什么

- 不是企业协同系统。
- 不是复杂权限后台。
- 不是传统文档列表。
- 不是每台手机都维护本地数据库再复杂同步。
- 不是商业笔记软件。

## 3. 核心用户

- 家里的主要记录者。
- 其他家庭成员：爸爸、妈妈、孩子、老人等。
- 使用设备以安卓手机为主。

## 4. 核心场景

- 随手记录家庭事务。
- 记录维修、售后、物业、水电、宽带。
- 记录购物消费、发票、保修。
- 记录老人健康、孩子教育、宠物事项。
- 导入 Note Station 历史记录。
- 从 NAS 统一备份和导出数据。

## 5. MVP 体验重点

第一阶段前端原型先使用 mock 数据，不接真实后端，不写数据库逻辑。

当前 V1 移动端原型已覆盖：

- 首页 / 生活时间线
- 快速新建记录页
- 记录详情页
- 搜索页
- 分类页
- 导入 Note Station 页
- 设置 / NAS 备份页

当前可点击交互已覆盖：

- 首页分类、成员和快速筛选。
- 搜索页关键词、分类、标签、成员和时间范围筛选。
- 分类页点击分类后回到首页筛选。
- 记录卡片进入详情页。
- 新建记录模拟保存。
- 设置页模拟 NAS 在线 / 离线和备份状态。
- 导入页模拟上传、解析预览、确认导入和完成状态。
- 设置页支持切换当前家庭成员，新建记录默认使用当前成员。

## 6. 多人和 NAS 设计原则

- 每条记录显示创建人。
- 首页支持看到家庭成员参与感，但不要有企业协作感。
- NAS 状态主要放在设置页，首页只做轻量保存状态。
- MVP 阶段家庭成员默认都能看全部记录。
- 网络断开时提示无法连接家庭 NAS。
- 多人冲突先提示刷新，不做复杂合并。

## 7. 第一阶段设计约束

- 以 7 张 V1 PNG 为唯一视觉参考。
- 不参考 V2。
- 不调用 Product Design。
- 不继续生成 PNG。
- 不一次性写完整大项目。
- 不接真实后端。
- 不提交复杂权限、同步、导入解析实现。

## 8. 后续转开发注意事项

- 继续保持移动端优先，不要把页面改成后台表格或企业协同界面。
- 当前交互全部是 mock 状态，下一阶段接后端时要保留同样的信息层级和卡片阅读节奏。
- 分类、标签、成员筛选已经形成前端交互模型，后续 API 可以围绕这些查询条件设计。
- Note Station 导入页目前只演示安全预览流程，真实解析必须等用户提供样例文件后再实现。
- 家庭成员体验保持轻量，默认全家可见；后续如做私密记录，应继续避免企业权限后台感。
